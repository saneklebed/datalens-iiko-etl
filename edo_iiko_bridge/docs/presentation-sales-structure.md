# Презентация для покупателя: мост ЭДО ↔ iiko

Структура слайдов/разделов для продажи. Можно перенести в PowerPoint, Google Презентации или оставить как текстовый документ.

---

## 1. Проблема

**Слайд: «С чем вы сталкиваетесь»**

- Входящие накладные и УПД приходят в ЭДО (Диадок), а учёт товара ведётся в iiko.
- Ручной перенос: открыть документ в ЭДО → вручную создать приход в iiko → легко ошибиться в количестве, цене, номенклатуре.
- Два окна, два интерфейса, дублирование работы, задержки при приёмке.
- Сложно контролировать: какие накладные уже внесены в iiko, по каким не хватает привязки номенклатуры.

**Ключевой месседж:** приходы из ЭДО должны попадать в iiko быстро, без ручного переписывания и с привязкой к вашей номенклатуре.

---

## 2. Решение

**Слайд: «Мост ЭДО ↔ iiko»**

- Единое окно внутри **iikoChain Office**: работа с входящими документами ЭДО и выгрузка приходов в iiko без переключения между системами.
- Интеграция с оператором ЭДО (получение документов, подписание, отказ) и с iiko (номенклатура, прайс-листы поставщиков, создание приходов).
- Привязка строк накладной к вашим товарам в iiko через прайс-лист поставщика — приход создаётся уже с вашей номенклатурой.

**Сейчас:** интеграция с **Диадок** и **iiko**.  
**В ближайшем будущем:** появится интеграция с оператором ЭДО **СБИС** — тот же функционал для клиентов, работающих через СБИС.

## 3. Функции (что умеет система)

**Слайд: «Функционал»**

### Настройки (вкладка в плагине)
- Подключение к Диадоку: логин, пароль, API-токен.
- Настройка создания накладных (включено/выключено).
- Все данные хранятся локально в iikoChain.

### Поставщики
- Список контрагентов из Диадока по выбранному юр. лицу.
- Колонки: организация, ИНН, КПП, краткие названия.
- Пагинация — доступны все контрагенты, не только первые 100.

### Накладные (входящие документы)
- Выбор юр. лица из Диадока.
- Фильтр по датам «С» / «По», кнопка «Получить накладные».
- Таблица: отправитель, ИНН, номер, дата, отправлен в ЭДО, сумма НДС, сумма, статус ЭДО, поставщик в iiko.
- Колонка **«Все привязки сделаны?»**: галочка — все строки привязаны к номенклатуре iiko, крестик — есть непривязанные (нужно допривязать).
- Автоматическое сопоставление отправителя с контрагентами; цветовая индикация (подписан — зелёный, отказ — розовый, поставщик найден — зелёный и т.д.).

### Детали накладной
- Просмотр строк УПД: наименование, артикул/код поставщика, количество, цена, сумма.
- Привязка строк к номенклатуре iiko по прайс-листу поставщика (код/артикул поставщика → товар iiko).
- Действия по документу:
  - **Подписать и выгрузить в iiko** — подписание в Диадоке и создание прихода в iiko.
  - **Отказать** — отправка отказа в подписании в Диадок.
  - **Выгрузить в iiko** — только создание прихода в iiko (если документ уже подписан или не требует подписи).
- Кнопка **«← Назад к накладным»** — возврат к списку.

### Массовые действия
- Выбор нескольких накладных в списке.
- Массовое «Подписать и выгрузить» / «Отказать» / «Выгрузить в iiko» для выбранных документов (с учётом статуса и привязок).

**Слайд: «Как это выглядит» (места под скриншоты)**

## 5. Выгоды для покупателя

**Слайд: «Что вы получаете»**

- **Скорость:** приход из ЭДО в iiko в несколько кликов вместо ручного ввода.
- **Точность:** привязка по прайс-листу поставщика — в iiko попадает ваша номенклатура, без путаницы в артикулах.
- **Порядок:** видно, по каким накладным привязки не завершены (крестики в колонке), что уже выгружено в iiko.
- **Один экран:** работа в iikoChain Office без постоянного переключения в личный кабинет ЭДО.
- **Подписание и отказ:** прямо из плагина — подписать и выгрузить или отказать контрагенту.
- **Масштаб:** массовые действия по нескольким накладным сразу.
- **Развитие:** поддержка и доработки — постоянная история; в планах — интеграция с оператором ЭДО СБИС.
