# Что положить в папку Plugins (iikoChain) — оболочка моста ЭДО ↔ iiko

**Назначение:** точка входа в iikoChain, которую видно глазами (пункт в меню / экран). Сначала — оболочка, потом наполняем логикой моста (Диадок, УПД, маппинг, iiko Resto).

**Целевая система:** iikoChain Office (не iikoFront). Всё завязано через плагины: в папку **Plugins** кладём нужные файлы.

**Важно:** у партнёрского EDI-Doc 3.02 в Chain Office есть **пункт в левой навигации** («EDI-Doc 3.02» → Документы, Настройки). Текущий наш плагин собран под **Resto.Front.Api** (API для касс) — он добавляет только кнопку в «меню дополнений» на экране дополнительных операций, **в навигацию Chain Office не попадает**. Чтобы получить пункт в навигации, как у EDI-Doc, нужен API/контракт расширения именно Chain Office; в открытой документации iiko его нет. Нужно уточнять у iiko (поддержка, партнёрская программа): как зарегистрировать свой пункт в навигации Chain.

---

## Куда копировать

**Путь iikoChain Office:**

```
C:\Program Files\iiko\iikoChain\Office\Plugins
```

Один плагин = **одна папка** внутри Plugins. Создайте подпапку **EdoIikoBridge** и положите в неё Manifest.xml и DLL (либо запустите скрипт `copy_to_chain.ps1` из папки с плагином).

---

## Что «накидать» в папку плагина

Минимум по контракту iiko (общая схема для плагинов):

| Что | Описание |
|-----|----------|
| **Manifest.xml** | Манифест плагина: имя dll, тип класса, версия API, идентификатор лицензии. Пример — в этой папке: `Manifest.xml.example`. |
| **DLL плагина** | Сборка плагина (.NET). Контракт плагинов iikoChain Office совместим с Resto.Front.Api (IFrontPlugin). Исходники и сборка — в `../EdoIikoBridgePlugin/`. |
| Дополнительно | Библиотеки, конфиги, ресурсы — по необходимости. |

После копирования перезапустить **iikoChain Office**, чтобы плагин подхватился.

---

## Связь с Python-мостом

Логика моста (Диадок, парсер УПД, iiko Resto, маппинг) живёт в **`edo_iiko_bridge/`** (Python, CLI). Оболочка в Plugins может:

- показывать экран/меню «ЭДО ↔ iiko» и вызывать внешний сервис или скрипты;
- либо со временем вызывать тот же функционал через API нашего сервиса.

Конфиги (URL моста, секреты) при необходимости хранить рядом с плагином или в настройках iiko.
