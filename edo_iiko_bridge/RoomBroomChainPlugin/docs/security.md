# Безопасность: защита DLL от копирования и реверса

Плагин попадает к покупателю в виде DLL на его компьютер. Чтобы тот не мог просто скопировать файлы и украсть продукт (прочитать логику, склонировать функционал), включена **многоуровневая защита** при сборке Release.

## Степени защиты (слои)

Конфиг обфускации: `RoomBroomChainPlugin.crproj`. При сборке Release применяются по порядку:

| Уровень | Защита | Что даёт |
|--------|--------|----------|
| **1** | **Name (preset normal)** | Переименование классов, методов, полей. В декомпиляторе — нечитаемые имена (a, b, c, .\u200b). |
| **2** | **Constants** | Шифрование строк и числовых констант. В коде не видно текстов сообщений, URL, ключевых слов. |
| **3** | **Ref proxy** | Вызовы методов идут через прокси. Сложнее проследить, кто кого вызывает и восстановить логику. |
| **4** | **Ctrl flow** | Поток управления запутан (switch-based). Восстановить порядок операций по декомпилированному коду трудно. |
| **5** | **Anti tamper** | Тела методов шифруются по checksum модуля. При изменении DLL модуль не загрузится. В декомпиляторе часто видны **пустые тела методов**. |
| **6** | **Anti debug** | Затрудняет отладку и профилирование (детект отладчика), усложняет пошаговый реверс. |

Публичный API плагина (точки входа для iikoChain) сохраняется, чтобы хост мог загрузить плагин.

**Если после добавления защит плагин перестал загружаться в iikoChain** (например, из-за unverifiable-кода после anti tamper), в `RoomBroomChainPlugin.crproj` можно закомментировать или удалить блок `<protection id="anti tamper"> ... </protection>` и пересобрать Release.

## Обфускация только в Release

- **Release** — все перечисленные защиты включены; в `dist/RoomBroomChainPlugin.zip` попадает обработанная DLL.
- **Debug** — обфускация не запускается; в `dist/RoomBroomChainPlugin-dev.zip` — обычная DLL для разработки и отладки.

Клиенту отдавать **только** `RoomBroomChainPlugin.zip` (для конечного пользователя).

## Два ZIP в dist

- **RoomBroomChainPlugin.zip** — для конечного пользователя (Release, все степени защиты). Только его отдавать клиенту.
- **RoomBroomChainPlugin-dev.zip** — для разработки (Debug, без обфускации). Удобно ставить себе для отладки и доработок.

Сборка по умолчанию создаёт оба архива:

```powershell
.\scripts\package.ps1
```

Один архив: `.\scripts\package.ps1 -Single -Configuration Release` или `-Configuration Debug`.

## Нет секретов внутри DLL

- Логин/пароль/API-токен Диадока и данные доступа к iiko **не зашиты** в код.
- Они хранятся в настройках iikoChain и в конфиге рядом с плагином.
- Даже при реверсе в DLL не будет паролей или ключей API.

## Дополнительные меры (по желанию)

- **Лицензирование:** привязка к ключу/организации (проверка при старте плагина). Копирование DLL на другой компьютер без лицензии не даст рабочего продукта.
- **Более жёсткий preset:** в `.crproj` можно поднять базовый preset до `aggressive` или `maximum` (предварительно проверить запуск в iikoChain).
- **Коммерческие протекторы:** .NET Reactor, Eazfuscator и т.п. — нативная упаковка и усиленная анти-отладка.

Итог: у покупателя на компе лежат DLL с **несколькими степенями защиты** — переименование, шифрование строк, запутанный поток управления, защита целостности и анти-отладка. Прочитать и украсть продукт по таким файлам существенно сложнее, чем по обычной сборке.
