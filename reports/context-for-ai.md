# AI-контекст — последние изменения

Этот файл помогает AI понять актуальное состояние проекта при смене устройства (дом/работа).

## Последнее обновление: 02.03.2026

### Плагин RoomBroom (edo_iiko_bridge/RoomBroomChainPlugin)

**Что работает:**
- Авторизация в Diadoc API (V3/Authenticate, HttpWebRequest)
- Загрузка юрлиц (GetMyOrganizations), контрагентов (GetCounteragents с пагинацией), документов (GetDocuments)
- Вкладка «Настройки» — сохранение логина/пароля/API-токена
- Вкладка «Документы» — три режима: Черновики, Контрагенты, Входящие
- Контрагенты: полный список с пагинацией, колонки Организация/ИНН/КПП
- Входящие: фильтр по дате, сопоставление документов с контрагентами (по BoxId + fallback по SellerInn), цветовая раскраска (Подписан=зелёный, Отказ=розовый)

**Что НЕ работает / не подключено:**
- Интеграция с iiko API (проверка поставщиков по ИНН, проверка накладных по номеру) — поля SupplierFound и IikoInvoice в модели есть, но пока не заполняются
- Вкладка «Черновики» — структура есть, функционал минимальный

**Ключевые грабли (не наступать повторно):**
- DateEdit в iikoChain: НЕ добавлять Buttons.AddRange (дублирует дефолтную кнопку). LabelControl рядом — обязательно задавать явный Size, иначе перекрывает DateEdit (z-order)
- BoxId из API приходит как `guid@diadoc.ru` — нужно StripBoxIdDomain
- GetCounteragents без пагинации возвращает только 100 — нужно пагинировать (afterIndexKey)
- Diadoc API V3/GetDocuments не возвращает имя отправителя и ИНН напрямую — нужно сопоставлять через контрагентов по CounteragentBoxId, ИНН брать из Metadata[Key=SellerInn]
- Проект собирается с DevExpress v16.2 (из iikoChain), а не v21.2 (из DLL Diadoc-плагина)
