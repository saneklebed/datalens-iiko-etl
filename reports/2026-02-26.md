## 1. Плагин «ЭДО ↔ iiko» для iikoChain Office

**Цель:** оболочка в iikoChain (не на фронтах), которую видно глазами; потом наполняем логикой моста.

- **Путь установки:** `C:\Program Files\iiko\iikoChain\Office\Plugins` (iikoChain Office, не iikoFront).
- **Проект плагина** — `edo_iiko_bridge/EdoIikoBridgePlugin/`:
  - C#, .NET Framework 4.7.2, Resto.Front.Api.V8.
  - Класс `EdoIikoBridgePlugin` (IFrontPlugin): кнопка «ЭДО ↔ iiko» в меню дополнений, по нажатию — AddWarningMessage (заголовок + текст про оболочку).
  - Manifest.xml, copy_to_chain.ps1 для копирования в папку Plugins после сборки.
- **Правки по ходу сборки в CI:** добавлен `using Resto.Front.Api`; колбэк AddButtonToPluginsMenu — один аргумент (кортеж), лямбда `_ =>`; AddWarningMessage(sender: string, message, null); путь вывода — AppendTargetFrameworkToOutputPath=false, в workflow проверка на подпапку net472.
- **Заготовка «что положить в Plugins»:** `edo_iiko_bridge/iiko_chain_plugins_folder/` (README + Manifest.xml.example), путь везде на iikoChain Office.

## 2. Интеграция с Квантом через API

- **Тянуться через API:** начали подключаться к Кванту не только на отправку (создание задач), но и на получение данных через API (тянуться).
- **Задачки по разным критериям:** создание задач в Кванте по разным критериям — помимо уже настроенных (коммуникация «Ознакомиться с результатами инвент», «ТОП недостач», «Результаты по ТОП недостач») добавлена/уточнена логика пушать задачки в зависимости от критериев (детали — в коде `alerts_bot.py`, секреты `KVANT_API_KEY`, `KVANT_ASSIGNEE_ID` в `alerts.yml`).
- Документация: `docs/kvant-integration.md`.

## 3. CI: сборка плагина без VS на других ПК

- **Workflow** `.github/workflows/build-chain-plugin.yml`:
  - Запуск: push в `edo_iiko_bridge/EdoIikoBridgePlugin/**` или workflow_dispatch.
  - Сборка на windows-latest (dotnet restore/build Release).
  - Артефакт **EdoIikoBridge-Plugin** (zip: EdoIikoBridge.Plugin.dll + Manifest.xml), retention 90 дней.
- **Установка на любой ПК:** скачать артефакт из Actions → распаковать → скопировать в `...\iikoChain\Office\Plugins\EdoIikoBridge\` → перезапустить Chain. Visual Studio и .NET SDK на целевом ПК не нужны.
- Связь workflow «Build iikoChain plugin» и «EDI-Doc bridge» не объединяли: разные триггеры и артефакты, оба зелёные.

## 4. Ограничение: пункт в навигации Chain как у EDI-Doc

- У партнёрского **EDI-Doc 3.02** в iikoChain Office есть **пункт в левой навигации** («EDI-Doc 3.02» → Документы, Настройки).
- Наш плагин собран под **Resto.Front.Api** (API для касс): он добавляет только кнопку в «меню дополнений» на экране дополнительных операций, **в левую навигацию Chain не попадает**.
- Чтобы получить пункт в навигации, как у EDI-Doc, нужен API/контракт расширения именно Chain Office; в открытой документации iiko его нет.
- **Зафиксировано:** в `edo_iiko_bridge/iiko_chain_plugins_folder/README.md` и в `reports/context-for-ai.md` — цель (пункт в навигации), ограничение (текущий плагин — Front API), следующий шаг (уточнять у iiko/партнёров).
