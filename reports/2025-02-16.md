- **Клиент Диадока:** авторизация DiadocAuth, GetMyOrganizations, GetDocuments (V3), GetEntityContent (V4). CLI: `fetch-incoming`, `fetch-document <messageId> <entityId>`.

- **Маппинг:** mapping_store (load/save JSON, find_mapping_for_line).

## 2. Интеграция с Kvant API (задачник)

**Подключение по API к Кванту** из alerts-бота: создание задач/коммуникаций после рассылки еженедельного отчёта в Telegram.

- **Документация:** `docs/kvant-integration.md`.
- **Секреты в GitHub:** `KVANT_API_KEY`, `KVANT_ASSIGNEE_ID` (прокидываются в `.github/workflows/alerts.yml`).
- **Код в `alerts_bot.py`:**
  - Конфиг: `kvant_api_key`, `kvant_assignee_id`.
  - Эндпоинт: `POST https://platform.kvant.app/openapi/tasks/store`, авторизация заголовком `api-key`.
- **Созданные задачи в Кванте (интересные):**
  1. **Коммуникация «Ознакомиться с результатами инвент»** — полный текст отчёта (сводка + по филиалам) уходит в Квант на assignee после успешной отправки в Telegram.
  2. **Задача «ТОП недостач»** — ТОП-2 по филиалам, проверка приходов, инструкция по ТК и ежедневному инвенту; срок 5 дней.
  3. **Задача «Результаты по ТОП недостач»** — сравнение прошлой и текущей недели по ТОП-2 недостач, фиксация прогресса; срок 24 часа.

Вызовы: после `send()` в Telegram в режиме once вызываются `send_kvant_test_message()`, `send_kvant_top_shortages_task()`, `send_kvant_top_shortages_results_task()`.

### Нарыто из DLL (ILSpy) по EDI-Doc 3.02 — как устроен визуал

- Пункт «**EDI-Doc 3.02**» в iiko появляется **из кода DLL**, не из конфигов.
- В `Diadoc.dll` найден `DiadocNS.ITSPlugin : INavBarPlugin, IPlugin`:
  - `MenuName = "EDI-Doc 3.02"`, `Version = "3.02"`.
  - `MenuGroup` возвращает два пункта (две вкладки): `TabPageFirst` и `TabPageSecond`.
  - Папка плагина/логов строится как `...\Plugins\EDI-Doc 3.02\` (через `AppDomain.CurrentDomain.BaseDirectory + "Plugins\\\\" + MenuName + "\\\\"`).
- Основной UI: `PageFirst : XtraUserControl` (DevExpress), вкладки **Черновики / Контрагенты / Входящие / Накладная**.
- Настройки: `Settings.Default.SettingsMain` (JSON `ConfigCL`), `Settings.Default.Save()`.
- В iiko создание накладной делает `ServerApi.Download_Invoice(Download_inv_Document)`, а сопоставления сохраняются через `IikoHiddenApi.SaveOrUpdateAnnouncements(...)`.
