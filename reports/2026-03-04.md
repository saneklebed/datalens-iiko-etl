## Интерфейс iiko — список входящих накладных

- **Колонка статуса накладной:** сузить ширину столбца «Статус накладной», чтобы он занимал только место под текст статуса и не «съедал» пространство под данные.
- **Колонки сумм:** уменьшить ширину столбиков «Сумма» и «Сумма НДС» — ориентир по значениям до ~2 млн (нет необходимости резервировать место под миллиардные суммы).
- **Колонка отправителя:** сузить столбик «От», оставив читаемость, но без избыточного свободного пространства.

## Плагин RoomBroom — входящие УПД и кнопки

- **Фильтр только по УПД:** во вкладке «Документы» в режиме «Входящие» ограничили список документов только УПД (исключили другие типы ЭДО‑документов, которые нам не нужны для прихода в iiko).
- **Кнопка «Обновить»:** добавлена отдельная кнопка обновления списка входящих — перезапрашивает документы за выбранный период без необходимости повторно задавать даты или переключать режимы.
- **Создание накладных «с проведением / без проведения»:** реализовано два варианта создания накладной из УПД — с проведением в iiko и без проведения (черновик). В UI добавлены отдельные кнопки/опции, чтобы явно выбирать сценарий.

## Плагин RoomBroom — раздел «Настройки»

- Собрана и оформлена отдельная вкладка «Настройки» плагина: логин/пароль, API‑токен Диадока, параметр «Создавать накладные сразу проведёнными» и прочие флаги поведения.
- Настройки сохраняются в локальное хранилище iikoChain и используются как при обращении к Diadoc, так и при создании накладных из УПД.

## Интерфейс iiko — карточка входящей накладной

- **Шапка с поставщиком:** отделить блок с кнопкой «Назад к входящим» от информационного блока о поставщике; информацию по поставщику выровнять по центру по вертикали и оформить отдельным, аккуратным блоком со строками:
  - «Наименование: …»
  - «ИНН: …»
  - «Номер и дата документа: …».
- **Табличная часть — номер строки:** сузить колонку с номером позиции (№ товара), чтобы она занимала минимально достаточную ширину.
- **Табличная часть — количественно-ценовые поля:** сузить столбики «Тара», «В таре», «Кол-во», «Цена», «Сумма», «Сумма НДС» по аналогии с макетами на скриншотах — без потери читабельности, но без лишнего «воздуха» по горизонтали.
