# Правило «Дневной отчёт» в Cursor

Этот файл — инструкция для участников: как завести в Cursor правило, чтобы по запросу получать краткий отчёт о сделанном за день (в рамках репозитория, по описанным ниже правилам).

---

## Как завести Rule в Cursor

1. Открой **Settings** (Mac: `Cmd + ,` или меню Cursor → Settings).
2. В поиске настроек введи **Rules** или открой раздел **Cursor** → **Rules for AI** (или **Rules**).
3. Нажми **Add Rule** / **New Rule**.
4. Заполни:
   - **Name** (название): например, `Дневной отчёт`.
   - **Content** (содержимое): вставь **текст правила** из блока ниже.
5. Сохрани. Правило будет действовать в твоём Cursor.

---

## Текст правила для вставки

Скопируй блок ниже целиком в поле **Content** при создании Rule в настройках (см. выше).

```markdown
---
description: Формирует краткий дневной отчёт на основе изменений в репозитории за сегодня.
globs: 
alwaysApply: false
---

# Дневной отчёт

Когда пользователь просит «дневной отчёт», «что сделал за день», «отчёт за день» или аналогично:

1. Проанализируй изменения за сегодня. **Изменения за сегодня = коммиты за сегодня + текущие незакоммиченные изменения** (working tree vs HEAD). Используй git-историю за сегодня и diff незакоммиченных файлов.
2. **Включай в отчёт:** изменения кода, правки в документации и конфигах, новые или обновлённые задачи.
3. **Не включай:** правки в личных или чужих проектах; автоформатирование без смысловых изменений.
4. Сформируй краткий список всех существенных действий за день (без жёсткого лимита пунктов). Сгруппируй пункты по типам работ:
   - **Код** — правки в коде, новые фичи, рефакторинг.
   - **Документация / конфиги** — правки в документации, конфигурации проекта.
   - **Тесты** — написание или правка тестов, тест-кейсы.
   - **Инфраструктура** — деплой, CI/CD, окружения, зависимости, сервера.
   - **Задачи** — новые или обновлённые задачи, статусы.
   - При необходимости добавь другие группы, если есть такие изменения.
5. Формат вывода:
   - **Дата:** DD.MM.YYYY
   - Затем блоки по группам с заголовком и списком пунктов под каждой группой.
   - При отсутствии изменений за сегодня: «Изменений за сегодня не было» или предложи посмотреть другой период.
6. Вывод выдавай готовым к копированию: его можно отправить руководителю или вставить в общий отчёт/чат.
7. Сохранять отчёт как файл с именем **YYYY-MM-DD.md** (например, `2026-02-16.md`) в папку Reports в свою подпапку (Никита, Станислав или Александр), если в репо есть такая структура.
```

Если в твоей версии Cursor нет frontmatter (`---`, `description`, `globs`, `alwaysApply`), вставь только часть после второй `---` (заголовок и нумерованный список).

---

## Как пользоваться

- В конце дня открой чат в Cursor (Composer или обычный чат) и напиши, например: **«Дневной отчёт»** или **«Что сделал за день»**.
- Cursor сформирует краткий список сделанного по описанным правилам (код, доки, тесты, инфраструктура, задачи; без личного и автоформатирования). Сохрани отчёт как **YYYY-MM-DD.md** в свою папку в Reports (Никита, Станислав или Александр), если она есть в репо. Либо скопируй результат и отправь руководителю.
